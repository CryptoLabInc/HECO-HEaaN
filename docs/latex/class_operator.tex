{\footnotesize\ttfamily bool Operator\+::supports\+Partial\+Evaluation (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

An operator is considered as non-\/partial evaluable if its not possible to apply the operator only on a few operands and then store the intermediate result to continue evaluation after more operands are known. \mbox{\hyperlink{class_for}{For}} example, 7 + a + 21 + 9 is partially evaluable to a + 37. But the expression 32 $<$ 193 $<$ a $<$ 12 would partially evaluate 32 $<$ 193 = true and then lead to true $<$ a $<$ 12 which is obviously not correct. Because of that we require that all operands are known for these kind of operators. \mbox{\Hypertarget{class_operator_a74ac2d3c1f1fe6c0f7562f993c643f72}\label{class_operator_a74ac2d3c1f1fe6c0f7562f993c643f72}}
